+ <p><strong>Ladder Results Updates<a name="ladderupdate"></a></strong></p>
+ <blockquote>
+   <p>As games complete the ladder results are NOT automatically updated. Individual user records are updated with win/loss information required but the results are not published. To do this an admin must issue the command &quot;/ladderactivate&quot;. As you might imagine this could be rather difficult on larger servers for an admin to stick around 24/7 issuing this command.</p>
+   <p>Unix admins can create the following shell script to do the work and execute as they see fit (hourly, daily, whatever) from CRON.</p>
+   <DIV class=postcolor>#!/bin/sh <br>
+       echo -e "<SPAN 
+ style="COLOR: red">username</SPAN>\n<SPAN 
+ style="COLOR: red">password</SPAN>\n/ladderactivate\n/quit\n" | nice -n 19 /usr/local/bnetd/bin/bnchat <SPAN style="COLOR: red">serverIP</SPAN> &gt;&gt; logfile<BR>
+   <BR>
+   The entry above should be all on the same line...<BR>
+   Replace username, password, serverIP with the appropriate values.
+   <!--IBF.ATTACHMENT_11012-->
+   </DIV>
+   <p>Windows admins are not so fortunate. Someone who is skilled at windows batch files and passing commands into programs may be able to assist. If you are able to assist here, please post your working solution to the forums, <a href="http://forums.pvpgn.org">http://forums.pvpgn.org</a></p>
+   <p>A final note. Those running larger systems may need to be aware of system CPU usage when issuing the /ladderactivate command. If CPU usage is excessive or causes degraded performance on other processes you may be forced to reduce the frequency at which you run it. Be nice and advise your users of the frequency that ladder updates are performed. </p>
+ </blockquote>
